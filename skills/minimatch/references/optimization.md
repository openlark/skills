# minimatch Optimization Levels

The `optimizationLevel` controls the degree of optimization performed on a pattern before parsing.

## Level 0

No optimization performed. `.` and `..` are preserved in the pattern, meaning they must appear in the same positions within the path.

```
a/*/../c matches a/b/../c, but does NOT match a/c
```

## Level 1 (Default)

Removes `..` that follow non-`**`/`.`/`..`/`''` parts:

```
./a/b/../*  →  ./a/*
```

Matches `./a/c`, does NOT match `./a/b/../c`. Preserves `.` and empty path segments.

## Level 2+ (Aggressive Optimization, Suitable for File Traversal)

Performs additional optimizations, **suitable for file traversal scenarios like glob**:

1. **Remove `..`**: Removes `..` that follow non-`**`/`.`/`''` parts, also removes safe empty paths and `.` (not at start/end/second position after `/`)
2. **`/**/../` transformation**: `<pre>/**/../<p>/<rest>` → `<pre>/{..,**}/<p>/<rest>` (where `<p>` is a part that does NOT contain `.`/`..`/`**`/empty)
3. **Deduplicate `**`**: `{a/**/b, a/b}` → `a/**/b`
4. **Deduplicate `*`**: `{a/{*,x}/b}` → `a/*/b`

### Important Limitation

After Level 2 optimization, `Minimatch.match()` still works correctly because it also optimizes the path string, but the RegExp generated by `Minimatch.makeRe()` **may not correctly match strings that have not been processed by `levelTwoFileOptimize()`**.
